C# Suction Utility

The suction program extracts source code for a given type-set from a source tree containing C# code and additional resource files. It uses the powerful Microsoft.CodeAnalysis package which provides abstract syntax tree (AST) and symbol information on the code to produce a vareity of outputs types, called emits, like compiled assemblies, project files or documentation.

Features

  • Extraction and reorganization of C# projects based on namespaces
  • Output assembly or project and solution files
  • Package sources for GIT publishing
  • Support for NuGet packages
  • Documentation generation based on Roslyn AST
  • Convert C# to TypeScript

Using Suction

Command Line

To extract a set of types, and their dependencies from a source tree in the working directory, just specifiy the type names as goals:

suction -g Class1 Class2

The names don't need to be qualified; wildcards '*' and '?' can be used.

For more elaborate construction, creating a suction-file may be useful:

<Suction xmlns="cfs:suction-1589"> <SourceDirectory Directory="Source"> <Source Filter="*.cs">Component1</Source> <Source Filter="*.cs *.html">Component2</Source> </SourceDirectory> <Goal> Class1 Class2 </Goal> <Output Name="suction" Kind="ConsoleApplication" Version="1.2.3.4" /> <Emit Destination="doc" Type="documentation" EntryPage="Index" ProductName="Some Product" /> </Suction>

Programming Suction

Core functionality is provided by the Suction class; it may be addressed directly or by means of the CSharpSuction.Program, which provides a command line interface together with a configuration file format, given by the ProjectDescriptor class.
